# vibecleaner 모델 선택 및 ONNX 추가 가이드

[한국어](model-guide.ko.md) · [English](model-guide.md) · [사용자 안내로 돌아가기](../README.md)

이 문서는 기본 모델의 차이와 직접 준비한 ONNX 모델을 추가하는 방법을 설명합니다. 처음 사용하는 경우에는 초기 설정과 설정 화면에서 `(권장)` 항목을 그대로 선택하면 됩니다.

## 기본 권장 조합

| 단계 | 권장 모델 | 용도 |
| --- | --- | --- |
| 감지 | RT-DETR-v2 FP32 | 말풍선과 대사 영역 찾기 |
| OCR | PP-OCRv6 Medium ONNX | 일본어를 포함한 문자 인식 |
| 인페인팅 | AOT ONNX | 원문을 지운 영역 복원 |

모델이 없으면 설정을 저장할 때 앱이 필요한 파일을 확인하고 다운로드합니다. 모델 파일은 설치 폴더가 아니라 `%LOCALAPPDATA%\vibecleaner\models`에 저장됩니다.

## 선택 가능한 모델

### 감지

- **RT-DETR-v2 FP32(권장)**: 기본 선택입니다. 정확도를 우선할 때 적합합니다.
- **RT-DETR-v2 INT8**: 다운로드와 실행 부담을 줄이고 싶을 때 사용합니다.
- **YOLOv8/11 ONNX**: 호환되는 YOLO 계열 모델을 사용할 때 선택합니다.

### OCR

- **PP-OCRv6 Medium ONNX(권장)**: 일반적인 만화 페이지와 세로·대각선 대사에 사용합니다.
- **PP-OCRv6 Small ONNX**: 자원 사용량을 줄이고 싶은 환경에 적합합니다.

OCR은 PP-OCR DB Detection과 CTC Recognition 파일을 함께 사용합니다. 이미지나 저신뢰 영역을 외부 Vision 서비스로 보내 OCR을 보완하지 않습니다.

### 인페인팅

- **AOT ONNX(권장)**: 일반적인 말풍선과 대사 제거에 균형이 좋습니다.
- **LaMa ONNX**: 넓은 영역이나 반복 무늬를 복원할 때 비교해 볼 수 있습니다.

결과가 말풍선 밖까지 번지면 `말풍선 경계 제한`을 켜고, 감지 범위가 잘못되었는지 고급 감지 옵션의 박스 표시로 먼저 확인하세요.

## 직접 준비한 ONNX 모델 추가

앱을 종료한 뒤 아래 구조로 파일을 복사하고 다시 실행하거나 설정 창을 다시 여세요.

```text
%LOCALAPPDATA%\vibecleaner\models\
├─ detection\
├─ ocr\
│  └─ 내-ocr-모델\
└─ inpainting\
```

### 감지 모델

`detection` 폴더에 ONNX 파일을 넣습니다. 파일명에 다음 계열명이 포함되어야 합니다.

- RT-DETR-v2: `rtdetr` 또는 `rt-detr`
- YOLOv8/11: `yolo`

YOLO 출력 클래스는 `0=말풍선`, `1 이상=텍스트` 순서를 사용해야 합니다. RT-DETR-v2 모델은 앱이 기대하는 labels, boxes, scores 출력 형식과 호환되어야 합니다.

### OCR 모델

`ocr` 아래에 모델별 하위 폴더를 만들고 다음 파일을 함께 넣습니다.

- 파일명에 `det`가 들어간 PP-OCR DB Detection ONNX
- 파일명에 `rec`가 들어간 PP-OCR CTC Recognition ONNX
- 모델 전처리와 문자 사전 정보가 담긴 `.yml` 또는 `.yaml` 파일

세 파일이 같은 폴더에 있어야 하나의 OCR 모델로 목록에 나타납니다.

### 인페인팅 모델

`inpainting` 폴더에 ONNX 파일을 넣습니다. 파일명에 `aot` 또는 `lama`가 포함되어야 합니다. 입력·출력 텐서 구조가 앱의 AOT 또는 LaMa 실행기와 호환되지 않으면 목록에는 나타나더라도 실행할 수 없습니다.

## 모델이 목록에 나타나지 않을 때

1. 확장자가 `.onnx`인지 확인합니다.
2. 파일명에 지원 계열명이 포함되어 있는지 확인합니다.
3. OCR은 det, rec, YAML 파일이 같은 하위 폴더에 있는지 확인합니다.
4. 설정 창을 닫았다 다시 열어 목록을 새로 불러옵니다.
5. 그래도 보이지 않으면 앱 로그에서 모델 검색 또는 ONNX Runtime 오류를 확인합니다.

사용자 모델은 파일 형식만으로 정확도나 완전한 호환성을 보장할 수 없습니다. 원본을 보관하고 소수의 페이지로 먼저 검증하세요.
