# ADR 0002: Use canonical bubble text layers

## Status

Accepted.

## Decision

VibeCleaner rasterizes each non-selected bubble into a cropped transparent PNG at page-native coordinates and DPR 1.0. A single offscreen Qt runtime shapes and rasterizes the text; the same cached pixels are displayed by the canvas and composited by Qt during export. The selected bubble alone is represented by an SVG editing overlay. The existing PNG remains visible until the replacement PNG or the editing font is ready, so transitions are atomic.

`QGuiApplication` and bundled-font registration are owned by the backend main thread. `QTextLayout`, `QGlyphRun`, `QRawFont`, `QImage`, and `QPainter` objects are created, cached, consumed, and destroyed by one render worker. Public caches contain only serializable geometry, PNG bytes, and digests.

Final crop and ink geometry come from the rendered alpha mask. Glyph outlines are only an inexpensive layout aid. Stroke is generated by expanding the rendered glyph alpha, then compositing the stroke and fill colors into an sRGB RGBA image.

## Resource identity

Tiles use a process-scoped, content-addressed URL:

```text
/api/text-layers/{namespace}/{page_id}/{bubble_id}/{cache_key}.png
```

A cached key is immutable. A current but evicted key is regenerated; an unavailable historical key returns `410 TEXT_LAYER_EVICTED`. Restarting the backend changes the public namespace and prevents stale WebView cache reuse.

The layout fingerprint covers text, geometry, masks, layout policy, font/style and shaped glyph output. The paint fingerprint adds fill, derived stroke, alpha expansion, PNG and color-space contracts. Tests compare decoded RGBA pixels and pixel digests instead of relying on cross-version PNG encoder byte determinism.

## Mutation and recovery

Visual mutations require both `expected_project_generation` and `expected_visual_revision`. They return changed-bubble patches. User content is committed even if a derived PNG cannot be produced; that bubble enters `fallback` and uses the editing renderer. Explicit recovery changes `fallback` to `ready` and increments the page visual revision. Recreating bytes for the same reference does not.

New-project and project-load operations clear process-local render caches. Export fails explicitly if a canonical text layer cannot be generated rather than silently drawing text through another renderer.
